AWS VPC: Subnets

A subnet is a logical subdivision of an IP network. Within a VPC, subnets provide a range of IP addresses for your resources, ensuring they are logically separated and isolated. They are tied to a single Availability Zone.

Key Concepts:

  • - CIDR Block: A subnet has a specific CIDR range (e.g., 10.0.1.0/24).
  • - Availability Zones: Subnets exist within a single AZ.
  • - Public vs. Private: Public subnets can access the internet, while private ones cannot (by default).

AWS VPC: Route Tables

A route table contains a set of rules, called routes, that are used to determine where network traffic from your subnet is directed. Each subnet in your VPC must be associated with a route table.

Key Concepts:

  • - Destination: The destination CIDR block (e.g., 0.0.0.0/0 for internet).
  • - Target: The gateway, interface, or connection to which the traffic is sent.
  • - Main Route Table: The default route table for the VPC.

AWS VPC: Security Groups

Security Groups act as a virtual stateful firewall for your EC2 instances to control inbound and outbound traffic. They operate at the instance level and only contain 'allow' rules.

Key Concepts:

  • - Instance Level: Attached to EC2 instances.
  • - Stateful: Responses to allowed inbound traffic are automatically allowed for outbound traffic.
  • - Default-Deny: All traffic is implicitly denied unless explicitly allowed by a rule.

AWS VPC: NACLs

A Network ACL (NACL) is a stateless firewall for controlling traffic in and out of a subnet. NACLs contain both 'allow' and 'deny' rules and process rules in numerical order, from lowest to highest.

Key Concepts:

  • - Subnet Level: Controls traffic for an entire subnet.
  • - Stateless: Separate rules are needed for inbound and outbound traffic.
  • - Rule Numbering: Rules are processed sequentially (e.g., rule 100 is checked before rule 200).
