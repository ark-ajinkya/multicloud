AWS Elastic Load Balancing

An interactive guide to automatically distributing your incoming application traffic across multiple targets, such as Amazon EC2 instances, containers, and IP addresses, for enhanced scalability and high availability.

Core Purpose of a Load Balancer

At its core, a load balancer serves as the "traffic cop" for your applications. It sits in front of your servers and routes client requests across all servers capable of fulfilling those requests in a manner that maximizes speed and capacity utilization and ensures that no one server is overworked.

Traffic Distribution

Automatically distributes incoming traffic across multiple targets, like EC2 instances or containers.

High Availability

Ensures no single point of failure by routing traffic only to healthy, operational targets.

Scalability

Enables your application to scale resources up or down based on demand without manual intervention.

Enhanced Security

Acts as a single, secure point of contact for external traffic, simplifying SSL/TLS management.

Application Load Balancer (ALB)

Layer 7

The Application Load Balancer is the ideal choice for modern applications that rely on HTTP and HTTPS traffic. It operates at the application layer (Layer 7), enabling it to make intelligent, content-based routing decisions. This means it can inspect the content of a request—such as the URL path, hostname, or HTTP headers—and direct the traffic to the appropriate backend service. This capability is essential for microservices architectures, where a single domain might serve many independent services.

Network Load Balancer (NLB)

Layer 4

When raw performance and ultra-low latency are paramount, the Network Load Balancer is the answer. Operating at the transport layer (Layer 4), it can handle millions of requests per second while maintaining a consistent performance profile. NLB is optimized for TCP, UDP, and TLS traffic where the destination is determined by IP address and port. A key feature is its ability to preserve the client-side source IP, which is critical for applications that need this information for logging, geolocation, or IP-based whitelisting.

Gateway Load Balancer (GLB)

The Gateway Load Balancer provides a unique service: it allows you to deploy, scale, and manage fleets of third-party virtual network appliances like firewalls, intrusion detection/prevention systems (IDS/IPS), and deep packet inspection systems in a transparent, highly available way.
