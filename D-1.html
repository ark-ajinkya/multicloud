Anatomy of a Command

Every shell command, no matter how complex, follows a basic structure. Understanding these three core components is the first step to fluency.

  • Command: The action to perform (e.g., `ls`).
  • Options: Modifiers that change the command's behavior (e.g., `-l`).
  • Arguments: The target of the command (e.g., `/home/user`).

Power Tools: Pipes and Grep

Pipes (`|`) are one of the shell's most powerful features. They let you chain commands, sending the output of one command directly as the input to another. This is often combined with `grep` to filter for specific information.

Example: Finding a Specific Process

This finds all running processes, then filters the list to show only lines containing "chrome".

ps aux

List all processes

|

grep "chrome"

Filter for "chrome"

Process Termination Signals

The `kill` command doesn't just "kill." It sends signals to processes. Different signals request different kinds of termination, from a polite request to a forceful shutdown. The `KILL` signal is the most powerful and should be used as a last resort.
